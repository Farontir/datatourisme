## üîß Development Guidelines

### Using Components
```tsx
import { Button, Card, CardContent } from '@datatourisme/ui';

function MyComponent() {
  return (
    <Card>
      <CardContent>
        <Button onClick={() => console.log('clicked')}>
          Click me
        </Button>
      </CardContent>
    </Card>
  );
}
```

### Component Props
All components follow consistent patterns:
- **Variant props**: `variant`, `size`, `color`
- **State props**: `disabled`, `loading`, `error`
- **Event props**: `onClick`, `onSubmit`, `onChange`
- **Accessibility props**: `aria-label`, `aria-describedby`, `role`

### Styling
Components use:
- **Tailwind CSS**: Utility-first styling
- **CVA (Class Variance Authority)**: Type-safe variant props
- **CSS Custom Properties**: Theme customization
- **Responsive Design**: Mobile-first approach

## üì± Responsive Design
